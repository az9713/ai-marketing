# Cialdini's 7 Levers - Implementation Checklist

## Authority Checklist

### Must-Haves
- [ ] Expert endorsement or quote
- [ ] Credentials visible (certifications, awards)
- [ ] Press/media logos ("As seen in...")

### Nice-to-Haves
- [ ] Founder bio with relevant credentials
- [ ] Industry recognition badges
- [ ] Published research or data
- [ ] Speaking engagements or conference mentions

### Implementation Examples
```
"Recommended by Y Combinator"
"SOC 2 Type II Certified"
"As featured in TechCrunch, Wired, Forbes"
"Built by ex-Google engineers"
```

---

## Social Proof Checklist

### Must-Haves
- [ ] Specific user/customer count
- [ ] Star rating (4.5+ stars)
- [ ] At least 3 testimonials

### Nice-to-Haves
- [ ] Named testimonials with photos
- [ ] Video testimonials
- [ ] Case studies with metrics
- [ ] Logo wall of recognizable customers
- [ ] Real-time activity feed

### Implementation Examples
```
"Trusted by 50,000+ developers"
"4.8 stars from 2,000+ reviews"
"See how Stripe reduced deploy time by 73%"
[Logo wall: GitHub, Stripe, Shopify, Notion]
```

---

## Commitment & Consistency Checklist

### Must-Haves
- [ ] Multi-step onboarding
- [ ] Progress indicators
- [ ] Value before payment

### Nice-to-Haves
- [ ] Personalization during signup
- [ ] Achievement unlocks
- [ ] "You've already..." reminders
- [ ] Setup investment before trial ends
- [ ] Data import early in flow

### Implementation Examples
```
"Step 2 of 4: Customize your workspace"
"You've already set up 3 integrations"
"Your personalized dashboard is almost ready"
"Complete your profile to unlock features"
```

---

## Liking Checklist

### Must-Haves
- [ ] Clear brand personality
- [ ] Visual design matches audience
- [ ] Relatable language/tone

### Nice-to-Haves
- [ ] Founder story
- [ ] Team photos
- [ ] Behind-the-scenes content
- [ ] Shared values statement
- [ ] Cultural references audience gets

### Implementation Examples
```
"Built by developers, for developers"
"We've been in your shoes"
[Casual, emoji-friendly tone for consumer apps]
[Technical, precise tone for enterprise]
```

---

## Reciprocity Checklist

### Must-Haves
- [ ] Free value before asking for anything
- [ ] Generous free tier or trial

### Nice-to-Haves
- [ ] Free tools/calculators
- [ ] Educational content (guides, courses)
- [ ] Templates or starter kits
- [ ] Free audit or assessment
- [ ] Unexpected bonuses

### Implementation Examples
```
"Free ROI calculator - no signup required"
"Get our 50-page guide to [topic]"
"Start free, no credit card required"
"Bonus: Free migration from [competitor]"
```

---

## Scarcity Checklist

### Must-Haves
- [ ] Clear, specific deadline or limit
- [ ] Believable scarcity (not fake)

### Nice-to-Haves
- [ ] Countdown timer
- [ ] "Only X spots/seats left"
- [ ] Early adopter pricing
- [ ] Beta access limits
- [ ] Seasonal offers

### Implementation Examples
```
"Early adopter pricing ends January 31"
"Beta limited to first 500 users"
"This offer expires in 48 hours"
"Only 3 enterprise seats remaining this quarter"
```

### Red Flags to Avoid
- Fake scarcity for infinite digital goods
- Countdown timers that reset
- "Only 2 left!" for SaaS subscriptions

---

## Unity Checklist

### Must-Haves
- [ ] Community or group to join
- [ ] Shared identity language

### Nice-to-Haves
- [ ] "Us vs them" positioning
- [ ] Private community access
- [ ] Member-exclusive content
- [ ] Movement language
- [ ] Shared enemy defined

### Implementation Examples
```
"Join 10,000 developers who refuse to use legacy tools"
"Built for founders, not enterprises"
"The indie hacker's alternative to [Big Corp]"
"Welcome to the future of coding"
"Leave bloated enterprise software behind"
```

| 7-8 | Strong | Well-implemented |
| 9-10 | Excellent | Best-in-class execution |
